# Bemo 功能路线图

> 参考 MemoFlow 功能列表，结合 Bemo 架构特点筛选整理。

## ⭐ 推荐加（近期）

- [x] **随机漫步** — 从笔记列表中随机取一条展示，实现简单，知识回顾价值高
- [x] **任务列表渲染** — 支持 `- [ ]` / `- [x]` 语法，开启 marked 的 GFM 扩展即可
- [x] **GFM 扩展渲染** — 删除线、表格、脚注、高亮等，marked 配置即可开启
- [x] **深色模式** — CSS 变量体系已就位，只需一套新的 `:root` 变量
- [ ] **版本历史** — 编辑前备份旧内容为 `.v1.md`，实现成本低，价值高
- [ ] **双链引用** — 类 Obsidian 的 `[[笔记名]]` 语法，知识管理核心特色功能
- [ ] **AI 总结/洞察** — 接入 OpenAI 或本地 LLM，对选定时段笔记做总结归纳
- [ ] **语音转文字** — 接入 Whisper 等 STT 服务，可与 AI 功能体系一起做

## ⏳ 后期可选

- [ ] **LaTeX 公式** — 需引入 KaTeX 库，仅数学/公式笔记需要
- [ ] **语音录制** — 前端 MediaRecorder API + 后端音频文件存储
- [ ] **音频/视频播放** — 内嵌播放器，配合语音录制或 Flomo 语音附件
- [ ] **多端同步（Server + WebDAV）** — 统一同步核心，同时支持用户自部署服务端和坚果云/Nextcloud WebDAV，详见 `SYNC_ARCHITECTURE.md`
- [ ] **Webhook 管理** — 为"微信输入"等外部接入预留的扩展能力
- [ ] **自定义快速工具** — 快捷输入模板、常用标签等
- [ ] **多语言 (i18n)** — 除非有国际化需求，当前中文界面已够用
- [ ] **拍照上传** — Capacitor 移动端打包后才需要

## ❌ 不需要

> 以下功能不适合 Bemo 的定位（本地优先、单用户、Web 端），不纳入计划：
> 登录认证、Explore 浏览、提醒系统、通知中心、应用锁、小组件、本地库模式

好的！我结合 Bemo 目前的架构（Python 后端 + Vue 前端 + 本地文件存储、无数据库）来逐条分析 MemoFlow 的功能对我们的价值：

🔐 用户与认证
功能 建议 理由
PAT 登录 / 账号密码登录 ❌ 不需要 Bemo 是本地部署的个人笔记工具，没有多用户系统，不需要认证。如果后续要做远程访问，可以考虑简单的 Token 鉴权，但不必复杂。
查看/编辑账号信息 ❌ 不需要 同上，单用户不需要。
Webhook 管理 ⏳ 后期可选 如果将来做"微信输入"或"AI 洞察"可以考虑 Webhook 接入，但优先级很低。
API 版本探测 ❌ 不需要 Bemo 是自己的后端，不存在版本兼容问题。
📝 Memo 基础
功能 建议 理由
新建/编辑/搜索/置顶/归档/删除 ✅ 已有 Bemo 已经实现了大部分（新建、编辑、搜索、置顶、删除、回收站）。归档可以后续加，相当于一个"隐藏但不删除"的中间状态。
标签过滤 ✅ 已有 侧边栏已实现标签点击筛选。
Explore 浏览 ❌ 不需要 这是 Memos 的多用户社区功能（浏览其他用户公开的笔记），Bemo 单用户不需要。
📄 Markdown 与任务清单
功能 建议 理由
基础 Markdown 渲染 ✅ 已有 已用 marked 库实现。
删除线/任务列表/表格/脚注/高亮 ⭐ 推荐加 marked 支持 GFM 扩展，只需开启配置即可。任务列表（- [ ] / - [x]）对笔记场景非常实用。
LaTeX 公式 ⏳ 后期可选 除非您经常记数学/公式类笔记，否则优先级低。需要引入 KaTeX 库。
双链引用与关联跳转 ⭐ 推荐加 类似 Obsidian 的 [[笔记名]] 双链，对知识管理非常有价值。但实现复杂度中等，建议作为阶段五的特色功能。
任务事项进度条 ⏳ 后期可选 锦上添花，在笔记卡片上显示 3/5 已完成 这种统计。可以在任务列表做完后顺便加。
📎 多媒体输入与预览
功能 建议 理由
附件上传 / 图片预览 ✅ 已有 编辑器已支持图片上传和渲染。
拍照上传 ⏳ 移动端才需要 如果之后做 Capacitor 移动端包装，这个自然会需要。Web 端用文件选择器已足够。
语音录制 ⏳ 后期可选 需要前端 MediaRecorder API + 后端存储音频文件。有一定价值但优先级中等。
音频/视频播放 ⏳ 后期可选 如果支持语音录制或导入了 Flomo 的语音附件，就需要播放器。可以和语音功能一起做。
语音转文字 ⭐ 有价值但需外部 API 需要接入 Whisper 或其他 STT 服务。如果做 AI 功能体系可以一起考虑。
🔄 版本历史与回收站
功能 建议 理由
回收站（恢复/永久删除） ✅ 已有 已完整实现。
版本历史查看/恢复 ⭐ 推荐加 非常实用。可以用简单的方式实现：每次编辑前把旧内容备份为 .v1.md、.v2.md 文件。实现成本低，价值高。
⏰ 提醒系统
功能 建议 理由
Memo 提醒 / 勿扰 / 铃声震动 ❌ 不推荐 这更像是一个待办工具的功能（如 Todoist），和 Flomo 式的"记录灵感"理念冲突。如果您需要提醒，建议直接用系统日历联动，而不是在笔记工具里做。

功能 建议 理由
Markdown + ZIP 导出 ✅ 已有 刚刚完成。
Flomo 导入 ✅ 已有 刚刚完成。
Flomo 导出 ✅ 已有 刚刚完成。
ZIP 导入 ✅ 已有 刚刚完成。
💾 本地库模式 & WebDAV
功能 建议 理由
本地库模式（扫描本地文件夹） ❌ 不需要 Bemo 本身就是本地优先，数据直接存在文件系统。不需要额外的"本地库"概念。
多端同步（Server + WebDAV）⭐ 推荐加（中期） 不把“用户必须有服务器”作为前提，而是设计统一同步核心，同时支持用户自部署服务端和坚果云/Nextcloud 等 WebDAV。协议与后端解耦，详见 `SYNC_ARCHITECTURE.md`。
🤖 AI 与复盘统计
功能 建议 理由
AI 总结 ⭐ 推荐加 侧边栏已经有"AI 洞察"占位。可以接入 OpenAI/本地 LLM，对选定时间段的笔记做总结归纳。高价值功能。
随机复盘 ⭐ 推荐加 侧边栏已有"随机漫步"占位。实现非常简单（从笔记列表随机取一条），但对知识回顾有意义。建议优先实现。
统计与热力图 ✅ 已有 热力图已实现。
🎨 体验与个性化
功能 建议 理由
主题（深色模式等） ⭐ 推荐加 CSS 变量体系已就位，加深色模式只需一套新的 :root 变量。成本低，体验提升大。
多语言 ⏳ 后期可选 除非有国际化需求，目前中文界面够用。
应用锁 ❌ 不需要 本地 Web 应用不需要应用锁，操作系统层面已有锁屏保护。
小组件 ❌ 不需要 这是原生移动端才有的能力，Web 端无法实现。
通知中心 ❌ 不需要 前面已建议不做提醒系统，通知中心也就不需要。
自定义快速工具 ⏳ 后期可选 如快捷输入模板、常用标签快捷键等，有一定价值但非核心。
📊 总结：推荐优先级排序
优先级 功能 预估工作量
P0 (简单高效) 随机漫步、任务列表渲染 各 0.5h
P1 (推荐) 深色模式、版本历史、GFM 扩展渲染 各 1~2h
P2 (中期) AI 总结/洞察、双链引用 各 4~8h
P3 (长期) 多端同步、语音录制+STT 需要架构设计
不推荐 登录认证、提醒系统、应用锁、小组件、Explore、本地库模式 -
